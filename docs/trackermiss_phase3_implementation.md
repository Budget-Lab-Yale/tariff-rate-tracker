## Architectural change in `06_calculate_rates.R`

The IEEPA reciprocal block (around lines 808–835, then 963–972) was previously
collapsing Phase 1 + Phase 2 + country-EO contributions into a single
`ieepa_country_rate` column, then applying the universal Annex II exempt
list to the merged total. The refactor preserves per-phase contributions
through the join, with two new columns:

- `country_eo_rate` — only the Phase 1 country EO (9903.01.76–.89)
  contribution. Subject to its own per-EO exempt list, NOT to universal Annex II.
- `country_eo_ch99` — the active EO ch99 code for this country (e.g.,
  `9903.01.77` for Brazil = census 3510). Used to look up per-EO exempts.

The `case_when` for `rate_ieepa_recip` now applies:
- **Universal Annex II** to the non-country-EO portion only:
  `if_else(is_universally_exempt, 0, ieepa_country_rate - country_eo_rate)`
- **Per-EO exempt list** to the country-EO portion:
  `if_else(is_country_eo_exempt, 0, country_eo_rate)`

Backward compatibility:
- Countries with no country-EO entry (most countries): `country_eo_rate = 0`,
  `country_eo_ch99 = NA` — the case_when reduces to the original behavior.
- Floor-rate countries (EU, Japan, Korea, Switzerland): `country_eo_rate = 0`
  by construction (none have country-EO entries in 9903.01.76–.89), original
  floor logic preserved.
- The `floor_exempt` and `floor` branches are unchanged.

## Section 201 wiring

Mirrors the Section 122 pattern: extract rate from Ch99 (here, presence-test
on 9903.45.21–.29), load product list (`s201_solar_products.csv`), apply
configured rate to all countries except Canada. New code lands at step 6b1
in `06_calculate_rates.R`.

The configured rate (14.5%) is used instead of the HTS `general` field
because the HTS shows the original Year-1 (2018) out-of-quota rate (30%),
which the proclamation explicitly steps down annually via US Note 18. The
HTS does not store the annual rate.

Solar 201 is also TRQ-structured (12.5 GW free, then the safeguard rate on
imports above quota). We don't currently model TRQ utilization — the
configured rate is best understood as a blended approximation. For more
accurate modeling, a TRQ-share parameter could be added.

## Validation

Pre-fix vs post-fix on the diagnostic cells (rev_26, 2025-10-15):

| Cell | Pre-fix | Post-fix | Census-implied | Status |
|---|---:|---:|---:|---|
| Brazilian coffee 0901.11.00.25 (3510) | 0% | 40% | 49.97% | **Recovered $170M+** |
| Brazilian coffee roasted 0901.21.00.20 | 0% | 40% | 36–38% | Recovered |
| Brazilian iron ore 2601.11.00.30 | 0% | 10% (universal baseline; iron ore on Brazil exempt list pre-Nov-13) | low | OK |
| Indonesian PV cells 8541.43.00.10 (5600) | 0% | 14.5% (Section 201) | 14–19% | Recovered |
| Vietnamese mics 8518.30.20.00 (5520) | 0% | 0% (Vietnam has no country EO; only Phase 2 which Annex II zeros) | 9–20% | Out of scope |

The Brazilian coffee cell is the smoking-gun validation. Census-implied 50%
breaks down as Phase 2 +10% (still zeroed by Annex II in our model) +
country EO +40% (now correctly applied). The remaining 10% gap is whether
Phase 2 inherits Annex II — left as a separate open question.

## Out of scope for this fix

1. **Vietnamese mics, palm oil, smartphone-style cells.** These are
   trackermiss because the universal Annex II suppresses Phase 2 for those
   products, and the country has no separate country EO to bypass it. Whether
   Phase 2 should inherit Annex II at all is an open policy interpretation —
   if not, that's a separate fix touching ~$11B of trackermiss but harder
   to justify legally.
2. **Switzerland watches at rev_6.** Likely a different bug (parser missing
   Switzerland from Phase 1 9903.01.69, or compound-duty handling). Worth
   a snapshot spot-check, ~$50M scale.
3. **TRQ modeling for Section 201.** The configured 14.5% rate is a blended
   approximation. Proper modeling would require quota utilization data.
4. **India 9903.01.84 country EO exempt list.** The scraper only handles
   Brazil. Other country EOs (India, plus 9903.01.76, .78–.89 if active)
   need their own annex extraction. Run `scripts/scrape_country_eo_annexes.R`
   after extending it to additional EOs.

## Reproducing

```bash
# Regenerate country-EO exempt CSV from cached Brazil annexes PDF
Rscript scripts/scrape_country_eo_annexes.R

# Rebuild rates from rev_25 onward (skips downstream daily/ETR/quality)
Rscript src/00_build_timeseries.R --start-from rev_25 --build-only

# Validate the diagnostic cells
Rscript scripts/validate_phase3_fix.R

# Regenerate ETRs config for one date and verify Brazil headline
Rscript src/generate_etrs_config.R 2026-01-01 ../Tariff-ETRs/config/2-21_temp/2026-01-01

# Optional: re-run compare_etrs.R (requires ETRs pipeline to re-run too)
```

  0.10 → ~0.40 (the Phase 2 portion remains suppressed by universal Annex II).

  Solar 201.

`tariff-etr-eval`'s trackermiss diagnostic should re-run against post-fix
snapshots. Expected behavior:

  to ~$30M (residual 10% gap).
- PV cell cells across Indonesia/Laos/Thailand drop substantially
  (Section 201 captured at 14.5%, leaving the 0–5% remaining gap as
  TRQ-utilization noise).
- Vietnamese mics, palm oil, French turbojet parts, etc. unchanged
  (handled by separate Phase 2-vs-Annex-II open question).
